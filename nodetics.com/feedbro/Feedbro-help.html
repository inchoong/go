服务插件

Feedbro为YouTube频道，YouTube Search，Instagram，Facebook，Twitter，VK.com，Yammer，Vimeo，Bitchute，SlideShare，Reddit，Pinterest，Rumble，Telegram，LinkedIn Groups，LinkedIn Home Feed和LinkedIn Job Search等多种流行服务提供内置支持。

只需导航到此类服务中的页面，单击Feedbro WebExtension图标，然后选择“在当前选项卡中查找源”以查看该页面是否受支持。

创建规则以筛选和管理文章

在“规则”菜单下，您可以定义自定义规则，这些规则在新文章到达或查看文章时进行修改。规则可以例如突出显示文章边框，添加标签，突出显示特定单词，显示桌面通知，播放声音，触发IFTTT操作，隐藏或删除文章。

按点对文章进行排序

在Feedbro中，您可以按日期（最新第一，最旧在前）或按点（如果文章具有相等的点，则大多数点在前，最新点在前）对文章进行排序。这可以使用Feedbro工具栏中的选择框进行调整。

要为文章分配分数，您需要在“规则”菜单下创建一个（或多个）规则。根据应添加的点定义条件，然后添加以数字为参数的操作“添加点”（例如 1）。此功能的强大之处在于，它允许您提高或“冒泡”重要的文章或提要。例如，如果您使用Feedbro关注100个YouTube频道，其中一些频道可能比其他频道更重要。通过创建为某些频道源提供比其他通道源更多的分数的规则，您可以按优先级顺序列出它们。

将源中的部分文章转换为全文文章

Feedbro具有内置引擎，用于将部分文本Feed文章转换为全文文章。此功能不会自动为所有源启用，必须为每个源启用。为此，请右键单击源树中的源，然后选择“属性”。然后使用“源条目内容”来调整全文提取设置。单击“预览”按钮以检查您是否获得所需的结果，然后按“保存”。

请注意，如果源已经提供了完整的文章，则显然不应使用全文选项。此外，全文转换不适用于所有源和网站，但它适用于大多数网站。

有用的第三方服务

kill-the-newsletter.com
kill-the-newsletter.com 允许您将来自特定电子邮件通讯组列表的电子邮件转换为可以使用Feedbro订阅的Feed。
google.com/alerts
Google快讯将帮助您监控几乎整个网络中与您的搜索条件匹配的内容，并提供您可以使用Feedbro订阅的Feed。
IFTTT.com
IFTTT（如果这样，那么那个）服务将允许您在Feedbro规则匹配时触发不同类型的操作（例如发送电子邮件或短信）。在 Feedbro 规则中选择 IFTTT 作为操作，并使用正确的 IFTTT Maker 渠道网址（例如 https://maker.ifttt.com/trigger/MYEVENT/with/key/AC11cggLDrvAWSZubQk7pn）作为参数（将 MYEVENT 替换为事件名称，将密钥替换为 IFTTT 提供的密钥字符串）。
mailnesia.com
Mailnesia提供免费的零设置一次性电子邮件地址，并支持RSS源。

设置 IFTTT 电子邮件通知操作

  1. 打开 ifttt.com 并登录
  2. 在 https://ifttt.com/create 创建新小程序
  3. 点击“+这个”
  4. 搜索“WebHooks”
  5. 单击“选择触发器”屏幕上的“接收 Web 请求”
  6. 在要求输入活动名称时输入 Feedbro
  7. 点击“+那个”
  8. 选择所需的操作（例如电子邮件）
  9. 单击“选择操作”屏幕上的“向我发送电子邮件”
  10. 如果需要，请修改电子邮件模板，然后单击“保存”。值 1 = 源条目链接，值 2 = 源条目标题，值 3 = 源条目作者
  11. 打开 https://ifttt.com/services/maker_webhooks/settings 并打开页面上提到的 URL。
  12. 从新页面中，您可以复制表单的实际Maker频道URL：https://maker.ifttt.com/trigger/ Feedbro/with/key/a1BYTM78EZcjzB1zjpQi8X
  13. 在 Feedbro 中创建或编辑规则
  14. 添加新的“IFTTT Maker Channel POST URL”操作，并将 Maker 频道 URL 作为参数输入到文本字段中
  15. 保存规则

设置不和谐频道规则操作

创建规则时，选择操作“不协调 HTTP POST URL”。

通过此操作，您可以将文章标题及其链接发送到Discord频道，您也可以使用手机实时监控该频道。参数是 Webhook URL，当您单击频道名称右侧的小“齿轮”图标（带有“编辑频道”工具提示）时，可以从 Discord UI 获取该 URL，单击“Webhooks”，然后单击“创建 Webhook”按钮。将Webhook命名为“Feedbro”，然后复制Webhook URL并保存Webhook。然后将 URL 粘贴到 Feedbro 规则操作参数字段。

杂项提示和技巧

  • 导航树中的源和文件夹可以通过拖放进行排列。
  • 右键单击任何源或文件夹以重命名、删除源项目或将源项目标记为已读。还可以更改源属性。
  • 您可以在“源统计信息”视图中轻松删除“死”源。
  • 单击“仅显示未读文章”图标可查看以前未看到的新文章。
  • 为“文章最长年龄”选择适当的值，以仅显示足够新鲜的文章。
  • 选择“列表视图”（快捷方式“2”）仅查看源条目标题的列表 - 对于快速浏览非常有用。
  • 单击列表视图中的文章顶部或底部以展开/关闭文章。
  • 定义规则以自动过滤/突出显示/隐藏/删除等Feed文章！
  • 将Feedbro URL拖放到书签栏，并在“选项”中的“所有项目”视图中启用启动，以便一键访问最新文章。
  • 如果您使用的是Linux，请安装Roboto字体以获得美观的UI（sudo apt-get install fonts-roboto）
  • 要获得干净的阅读视图，请隐藏导航器（快捷键“w”）并使用全屏模式 （F11）

常见问题

帮助！火狐不加载任何文章！
您的火狐个人资料很可能已损坏。发生这种情况时，IndexedDB API会停止工作，从而中断Feedbro。您可以使用此工具测试 IndexedDB API 功能。如果 IndexedDB 确实坏了，请打开 about：support 并点击“刷新 Firefox”。警告！这将删除所有附加组件及其设置，因此请先备份您的数据！
帮助！我有一些其他的技术问题！
请查看此列表以了解已知问题。如果您在那里找不到答案，请通过 feedbro.reader@gmail.com 与我们联系
如何仅查看未读文章？
启用“仅显示未读”模式，方法是单击工具栏中的星号+字母图标或按“u”，然后重新加载视图（单击树中的源或按“r”）。
使用查看模式 2、4 或 5 时，如何在 Feedbro 中查看文章？
不要点击文章标题。相反，请单击文章区域中的其他位置以展开Feedbro中的内容。
如何获取新文章的桌面通知？
要在新文章到达时收到桌面通知，您需要在“规则”菜单下创建新规则。选择“桌面通知”作为操作。
如何获得新文章的电子邮件通知？
首先创建一个包含 IFTTT.com Maker 频道的 Webhook，以便获取可用于触发电子邮件发送的 URL。然后在 Feedbro 中创建一个规则，其中包含 IFTTT Maker 渠道操作和该 URL 作为参数。
如何为每个 Feed 保存超过 100 篇文章？
右键单击源树中的源，然后选择“属性”。然后，您可以将“要保存的文章数”调整为2000。然后点击“保存”。请注意，最初大多数 XML 源仅提供最新的 10-20 篇文章。因此，在添加 Feed 时，您通常无法获取过去的所有文章。只有最新的。
如何让 Reddit 源正常工作？
您必须禁用Reddit的跟踪器保护，因为默认情况下Firefox会阻止所有XMLHttpRequests到Reddit。
首先打开 reddit.com 然后单击 URL 栏左侧的小盾牌图标。然后使用滑块禁用“此站点的增强跟踪保护已打开”。请参阅此图像。

如果这不起作用，请打开“关于：首选项#隐私”，然后在“增强的跟踪保护”下选择“自定义”，然后选择“仅在私人窗口中”作为“跟踪内容”选择框。
如何缩短 Feed 的更新间隔？
右键单击源树中的源，选择“属性”，然后调整“扫描间隔”设置。它可以低至1分钟。
如何从 Firefox 或 Chrome 导入我的书签？
打开“选项”页面并使用“导入火狐书签（作为OPML）”按钮。
如何预览源？
单击Feedbro扩展图标，然后选择“在当前选项卡中查找源”。然后单击蓝眼图标以预览所选源。
如何获取 Feed 的完整文章，而不是部分文章？
右键单击源树中的源，选择“属性”，然后使用“源条目内容”设置。选择“完整文章正文”或“带有文章主图像的完整文章正文”，然后单击“预览”以检查是否正确获取文章。然后点击“保存”。请注意，这只会影响在选择此设置后将加载的新文章。换句话说，已经加载的文章不受影响。
如何手动重新加载所有源？
要重新加载您手动拥有的所有源，请单击Feedbro工具栏左上角的蓝色重新加载图标。要重新加载特定文件夹下的所有源，请右键单击该文件夹，然后选择“更新文件夹中的源”
如何按作者对文章进行排序？
这是不可能直接实现的。但是，您可能希望仅关注/排序某些选定人员的文章，而不是每个人。您可以在“规则”菜单下创建一个规则，如果文章的作者是您想要关注的人之一，则该菜单会分配某个标签。如果需要，可以为每个作者创建一个新标记。
我如何在互联网上收到与我的搜索匹配的任何新文章的通知？
您可以使用 Google 快讯创建可生成 RSS 源的自定义搜索。然后在Feedbro中订阅它。
如何删除源？
右键单击源树中的源，然后选择“删除”。
如何创建子文件夹？
为了简单起见，Feedbro不支持子文件夹。
在哪里可以找到规则的音效？
有几个网站在互联网上托管免费的声音效果。其中之一是 freesound.org。如果在站点上单击“播放”之前打开“开发控制台”和“网络”视图并按媒体进行筛选，则可以获取声音剪辑 URL。
Feedbro有搜索功能吗？
目前Feedbro不支持全文搜索。原因是HTML5 IndexedDB引擎本身不支持它，虽然可以将全文搜索实现为一个层，但它会对性能产生重大影响。为不同的语言编写语言词干分析器也非常具有挑战性。

但是，您可以使用规则为文章分配标签。然后可以通过“标签”菜单访问标签，通过单击标签，您可以立即找到带有该标签的文章。
如何将Feedbro UI翻译成另一种语言？
目前，Feedbro支持开箱即用的英语和芬兰语。您可以在“选项”中选择语言。您还可以下载区域设置模板并创建新的翻译（请记住也要编辑元数据）。然后将翻译后的文件上传到例如github，并将翻译后的区域设置导入Feedbro。通过这种方式，您可以将Feedbro翻译成您想要的任何语言（帮助文件除外）。当人们将区域设置文件放到网上并共享链接时，每个人都会受益。如果您希望在Feedbro中包含已翻译的区域设置文件，请通过以下方式与我们联系 feedbro.reader@gmail.com

如果您在github上托管区域设置，请更新JSON中的元数据以将您的数据包含在“locale_author_email”和“locale_author_name”中，并确保“locale_source_url”指向实际的JSON文件（URL以 https://raw.githubusercontent.com/ 开头）。导入区域设置时，您还需要提供“原始”URL。

可以在此页面上找到现有翻译的链接。
如何将 YouTube 订阅内容下载为 OPML？
您可以在此页面上将 YouTube 订阅导出为 JSON。

收到YouTube订阅JSON文件后，您可以使用Feedbro选项页面上的“导入Feed订阅（作为OPML）”按钮将其导入。Feedbro会自动将文件转换为OPML，并在“YouTube”文件夹下导入您的订阅。
为什么不使用 chrome.storage.sync / Firefox sync 来同步 Feed 订阅和 Feed 输入数据？
不幸的是，Firefox也使用的chrome.storage.sync API对您可以存储的数据量有非常严格的限制。它还会限制每小时的 API 请求数。该 API 甚至无法提供足够的空间来存储 Feed 订阅数据，并且肯定不能用于处理 Feed 条目数据。
为什么选择框和输入字段在 Chrome 83+ 上看起来很丑陋？
Chrome更新了表单元素的默认折叠，这很丑陋。您可以在此处禁用它：chrome://flags/#form-controls-refresh
为什么 Twitter 集成不起作用？
Twitter 集成要求你允许 twitter.com 域设置 Cookie。请确保您没有阻止它。
为什么LinkedIn集成不起作用？
LinkedIn集成要求您允许 www.linkedin.com 域设置 Cookie。请确保您没有阻止它。
为什么Instagram集成不起作用？
Instagram可以暂时禁止您，如果您经常扫描多个个人资料而没有登录。在大多数情况下，登录Instagram应该防止被屏蔽。
